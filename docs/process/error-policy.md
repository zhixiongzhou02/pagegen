# 错误策略

这个文档定义错误怎么暴露、怎么记录、怎么定位。

## 分层原则

- 用户层：给出简洁、可执行的错误提示
- 开发层：记录足够定位问题的错误信息
- 文档层：说明常见错误属于哪一层

## 用户可见错误

- 不要只显示“失败了”
- 错误文案应尽量说明失败位置，例如：
  - 生成请求超时
  - 模型返回格式无效
  - 导出路径无效
  - 设置缺少 API Key

## 日志记录

- 外部接口失败应记录提供方、模型、阶段和错误消息
- 本地存储失败应记录路径和操作类型
- UI 侧错误至少要保留用户动作上下文

## 定位顺序

1. 看 UI 错误文案
2. 判断属于 UI、IPC、后端还是外部依赖
3. 再去对应模块和测试里定位

## 当前改进重点

- 让“页面生成失败”变成可区分的具体错误
- 让代理超时和模型格式错误在 UI 层可见
